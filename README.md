# 1️⃣ What is the difference between var, let, and const?

### Ans:- var is the old method; it works throughout the entire function and can be redeclared multiple times with the same name. let is the modern method; it works only inside a specific block ({ }) and its value can be changed. const is much like let, but its value cannot be changed once it is assigned. let and const are much safer because they do not work outside of their defined blocks.

# 2️⃣ What is the spread operator (...)?

### Ans:- The spread operator (...) is a brilliant feature in JavaScript that helps to spread or break down the elements of an array or object individually. It primarily works to "unpack" or extract data from a collection so that they can be used directly elsewhere. This helps to make the code much more concise and clean.

# 3️⃣ What is the difference between map(), filter(), and forEach()?

### Ans:- The main difference between map(), filter(), and forEach() in JavaScript lies in their results and purpose of use. map() works on every element of an array and creates a new array as a result. filter() selects elements from an array based on a specific condition and provides a new array containing only the elements that meet that condition. On the other hand, forEach() does not return anything; it is simply used to perform a specific action or operation on each element of the array.

# 4️⃣ What is an arrow function?

### Ans:- An arrow function is a shorter and more modern way to write functions in JavaScript that was introduced in ES6. It uses the "fat arrow" (=>) syntax, which allows you to skip the function keyword and even the return keyword if the code is only one line. This makes them very useful for keeping code clean and avoiding errors when working with callbacks or classes.

# 5️⃣ What are template literals?

### Ans:- Template literals are a modern way to handle strings in JavaScript using backticks instead of single or double quotes. They allow you to easily embed variables or expressions directly into a string using the syntax, which is much cleaner than using the plus sign for concatenation. This makes your code more readable and easier to write when dealing with complex text or dynamic data.
